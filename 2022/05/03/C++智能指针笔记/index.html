C++智能指针笔记

概述

本文整理了最近在学习C++智能指针源码时的一些笔记，代码摘自gcc4.9.2的源码，部分内容出自参考栏中引用的博客

特点

C++11中引入了unique_ptr，shared_ptr以及weak_ptr三种智能指针来帮助程序员动态管理内存。三者主要的特点分别是：

unique_ptr的特点

  • 独享被管理对象，当unique_ptr被销毁时，它所指向的对象也被销毁，是典型的RAII特性

  • 可以进行移动赋值和移动构造，但是不可以拷贝赋值和拷贝构造

shared_ptr的特点

  • 共享被管理对象，即可以有多个shared_ptr指向同一个对象，当最后一个shared_ptr被销毁时，被指向的对象也被销毁

  • 实现以上特点的核心机制是引用计数

weak_ptr的特点

  • weak_ptr是一种弱化版的shared_ptr，是为了解决shared_ptr的循环引用而引入的

  • 当有新增的weak_ptr指向shared_ptr指向的对象时，shared_ptr的引用计数不增加

  • 不拥有对象的所有权，但可以判断对象是否存在和返回指向对象的shared_ptr类型指针

移交所有权：reset以及release

虽然unique_ptr不能进行拷贝构造以及拷贝赋值，但是unique_ptr中提供了reset和release函数用于所有权的转移。二者的区别在于reset会释放掉原来绑定的对象，而release只是将旧的unique_ptr旧为空

release

unique_ptr中的release函数放弃对指针p的控制权，返回p并令内部_M_t中的指针为空

make_unique

除了构造函数和移动构造函数以外，C++14以后的标准库还提供了一个函数接口make_unique用于创建unique_ptr实例，这个make_unique是通过完美转发机制实现，类似vector的emplace_back

在Effecitve Modern Cpp的Item 21中提到优先使用std::make_unique和std::make_shared而不是直接使用new来创建智能指针，主要原因是减少代码的重复度（结合auto来实现这个目标）以及防止内存泄露，具体细节这里不展开分析。但是这个函数构造的unique_ptr有个缺点就是不能自定义delete函数，因此需要根据实际使用场景来决定创建智能指针的方式

shared_ptr和weak_ptr

weak_ptr是一种弱化版的shared_ptr，是为了解决shared_ptr的循环引用而引入的，关于循环引用可能发生的场景可参见这篇博客。

本小节重点在于剖析shared_ptr和weak_ptr在设计与实现方面的不同，主要参考自一篇对二者分析的很透彻的博客C++ 智能指针（shared_ptr/weak_ptr）源码分析

shared_ptr和weak_ptr的类关系

关于管理对象与被管理对象的说明：

  • 用new创建后托管给shared_ptr等智能指针管理的对象叫做被管理对象（managed object）

  • shared_ptr等智能指针内部创建的用来维护被管理对象生命周期的实例叫做管理对象（manager object）

共享被管理对象的shared_ptr也共享同一个管理对象

weak_ptr

weak_ptr的实现与shared_ptr类似，与shared_ptr主要不同的是weak_ptr的管理对象是__weak_count而不是__shared_count

换句话说，__shared_count内部的use_count主要用来标记被管理对象的生命周期，weak_count主要用来标记管理对象的生命周期
